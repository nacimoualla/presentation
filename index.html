Multi-Threading

& Synchronisation

Maîtriser l'exécution parallèle et éviter le chaos.

Qu'est-ce qu'un Thread ?

  • Un processus est une application en cours d'exécution.
  • Un thread est la plus petite unité de traitement.
  • Attention : Les threads partagent le même espace mémoire !

Pourquoi le Multi-Threading ?

↓ Utilisez les flèches du bas pour explorer (Slider) ↓

1. Réactivité ⚡

L'interface utilisateur ne gèle pas pendant les longs calculs en arrière-plan.

2. Performance 🚀

Exploitation maximale et simultanée des processeurs multi-cœurs modernes.

3. Économie 💾

Créer un thread coûte beaucoup moins de mémoire que de créer un nouveau processus.

Le Danger : Race Conditions

Que se passe-t-il si deux threads modifient la même donnée en même temps ?

  • -> Données corrompues
  • -> Comportements imprévisibles
  • -> Crash de l'application

La Synchronisation 🛡️

Mettre de l'ordre dans l'exécution avec des verrous.

Mutex

Verrou d'exclusion mutuelle.
Un seul thread à la fois.

Sémaphore

Limitation d'accès.
N threads maximum autorisés.

Moniteur

Approche Objet.
Méthodes synchronisées nativement.

Le Cauchemar : Deadlocks

L'Interblocage fatal.

"Le Thread A attend que le Thread B lâche une ressource... mais le Thread B attend que le Thread A lâche la sienne."

Résultat : Tout se fige. 🛑

Bonnes Pratiques ✅

  • Minimiser les données partagées entre les threads.
  • Garder les sections critiques (verrouillées) très courtes.
  • Toujours utiliser unlock() dans un bloc finally.
